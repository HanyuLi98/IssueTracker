# AE 任务管理平台

面向 Application Engineer 团队的全流程任务跟踪系统，覆盖售前咨询到售后维修的完整业务链路，数据存储于 Supabase。

---

## 技术栈

| 层 | 技术 |
|---|---|
| 前端框架 | React 18 + Vite |
| 数据库 | Supabase (PostgreSQL) |
| 存储 | Supabase Storage（发票 PDF） |
| 样式 | 内联样式（无 CSS 框架） |
| 字体 | Geist + Noto Sans SC |

---

## 功能模块

### 任务分类

| 模块 | 说明 | 关键字段 |
|---|---|---|
| 售前咨询 | 客户咨询记录 | 客户、对应销售、问题、状态、备注 |
| 售中任务 | 项目推进中的任务 | 客户、任务类型、问题、状态、备注 |
| Key Project | 重点大项目跟踪 | 客户、项目描述、工艺、需求、对应销售 |
| Ticket | 售后支持工单，含计时器 | 客户、序列号、问题描述、解决计时 |
| 质保内维修 | 保修期内维修记录 | 客户、售后单号、序列号、维修内容 |
| 付费维修及易损件 | 付费维修，含报价和 Invoice | 客户、序列号、报价、Invoice PDF |

### 状态流转

```
Ongoing → Pending → 已解决
```

标记为"已解决"时自动记录时间戳，Ticket 模块实时显示处理时长。

---

## 核心功能

### 全局隐藏/显示已解决
顶部 Tab 栏一键切换，对所有分类同步生效。

### 备注时间线
备注字段以日志形式追加，每次更新自动加日期前缀：

```
5.21 跟进了报价，客户表示需要内部审批
5.20 完成现场调试，待客户确认
```

双击备注单元格 → 输入本次更新内容 → 回车保存，旧内容自动保留。

### 连续天数新增
新增记录时可切换「单天 / 连续天数」，选择起止日期后批量创建，每天一条记录。

### 售前大项目快捷转入
售前咨询新增时勾选「标记为大项目」，填写项目描述 / 工艺 / 需求，保存后自动同步创建 Key Project 记录。

### 付费维修报价统计
付费维修页顶部实时显示：

- **报价总额** / **进行中** / **已解决** 三张统计卡
- Q1 / Q2 / Q3 / Q4 多选季度筛选，支持任意组合（Q1 + Q3 等），统计和表格联动更新

### 其他

- **双击行内编辑**：客户名称、问题描述、备注等字段双击直接编辑，无需进入表单
- **置顶**：重要记录钉在表格顶部
- **Ticket 迁移**：一键将 Ticket 迁移到质保维修或付费维修
- **周报导出**：按个人 / 区域 / 全员维度导出当周 CSV 周报
- **按季度 / 工程师 / 区域筛选**：侧边栏快速切换视图
- **操作日志**：记录所有增删改操作，可在侧边栏查看
- **翻译**：问题描述和备注字段一键中英互译
- **多主题**：深黑 / 午夜蓝 / 炭灰 / 明亮，可调字号

## 本地开发

```bash
# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 填写 VITE_SUPABASE_URL 和 VITE_SUPABASE_ANON_KEY

# 启动开发服务器
npm run dev

# 构建
npm run build
```

---

## 数据库初始化

在 Supabase SQL 编辑器中按顺序执行：

```
supabase-setup.sql        # 基础表结构
supabase-update-v4.sql    # 审计日志
supabase-update-v5.sql    # keyProject + 置顶字段
supabase-storage.sql      # Invoice 文件存储桶
supabase-add-pinned.sql   # 置顶字段补丁（如需）
```
